# PaaS & IaaS Doctrine
business_type: paas-iaas
sector: Technology & SaaS

## Visual Doctrine

### Colors
- primary: "#0A2540"
- secondary: "#0E1729"
- accent: "#00C2FF"
- psychology: Deep infrastructure navy conveys enterprise-grade stability and uptime, while an electric cyan accent signals speed, throughput, and developer-grade performance.

### Typography
- heading: IBM Plex Sans (Sans-serif)
- body: IBM Plex Mono (Monospace)
- psychology: A precise technical sans paired with monospace details signals engineering credibility and speaks the native language of the developer buyer.

### Imagery
- style: Architecture diagrams, terminal snippets, network topology visuals, and abstract node-and-edge graphics.
- subjects: Code blocks, dashboards, latency maps, and system architecture rather than people.
- psychology: Images should trigger trust in scale and the reassurance that the platform is built by people who understand the stack.

### Layout
- style: Documentation-adjacent layout with code samples, spec tables, and clear tiered pricing.
- white_space: Moderate
- psychology: Dense but well-structured information rewards technical evaluators who trust substance over polish.

### UI Patterns
- navigation: Persistent top nav with Docs, Pricing, and a "Start building" plus "Contact sales" dual path.
- cta_style: Two-track CTAs — self-serve "Deploy now" for developers and "Talk to sales" for enterprise procurement.
- psychology: Serving both the bottom-up developer and top-down buyer mirrors how infrastructure actually gets adopted.

### Copy Style
- tone: Precise, technical, proof-driven.
- length: Medium
- psychology: Specificity — benchmarks, SLAs, regions — builds the credibility that risk-averse infrastructure buyers require.
